# DayNavi 要求定義書

## 1. 背景

日々の予定管理では、大学の予定、就職活動、個人開発、学習予定、アルバイト、私用など、複数の予定やタスクを並行して管理する必要がある。

既存のカレンダーアプリでは、予定を日付や時刻に合わせて登録することはできる。しかし、「今日やるべきこと」「締切が近い予定」「就活で対応すべきタスク」などを、自分の行動に合わせて分かりやすく確認しづらい場合がある。

特に就職活動では、インターン応募、ES締切、面接日程、Webテスト、説明会、企業ごとのメモなど、管理すべき情報が多い。これらをカレンダー、メモアプリ、メール、就活サイトなどに分散して管理すると、締切漏れや準備不足につながる可能性がある。

また、予定を登録していても、朝や作業前にその日の予定を確認し忘れることがある。予定ごとに通知が来るだけでは、1日の全体像を把握しづらい場合もある。

そこで、予定・タスク・就活イベントを一元管理し、今日やるべきことを分かりやすく表示できるWebアプリを開発する。さらに、朝やユーザーが指定した時刻に、その日の予定をまとめてリマインドできる仕組みを目指す。

## 2. 現状の課題

現在の予定管理には、以下のような課題がある。

* 今日やるべき予定やタスクを一目で確認しづらい
* 朝に今日の予定を確認し忘れることがある
* 就活の応募締切や面接日程を忘れやすい
* カレンダー、メモ、メール、就活サイトなどに情報が分散しやすい
* 締切が近い予定を優先して確認しづらい
* 自分が確認したい時間に、その日の予定を把握しづらい
* 予定ごとの通知だけでは、1日の全体像を把握しづらい
* 予定とタスクが分かれており、実際の行動に落とし込みにくい
* 予定登録時に、日付・時刻・カテゴリ・メモなどを手動で整理する手間がある
* パスワードを忘れた場合に、自分で安全に再設定できる仕組みが必要である

## 3. 目的

本アプリの目的は、予定、タスク、就活イベントを一元管理し、ユーザーが今日やるべきことや締切が近い予定を分かりやすく確認できるようにすることである。

また、朝やユーザーが指定した時刻に、その日の予定をまとめて確認できるようにすることで、予定の確認漏れ、締切漏れ、準備不足を防ぐことを目的とする。

将来的には、AIによって自然文から予定やタスクを整理し、登録候補を提案する機能を追加することで、予定作成の手間を減らし、より使いやすいスケジュール管理を実現する。

## 4. 想定ユーザー

本アプリの想定ユーザーは以下の通りである。

* 就職活動中の学生
* インターンシップに応募している学生
* 大学の予定、学習、個人開発を並行して管理したい学生
* 日々の予定とタスクをまとめて管理したい人
* 既存のカレンダーアプリでは、今日やることが分かりにくいと感じている人
* 朝や決まった時間に、その日の予定をまとめて確認したい人

## 5. ユーザー要求

ユーザーは以下のことを実現したい。

* 予定を登録したい
* 日付ごとの予定を確認したい
* 今日の予定を一覧で確認したい
* 朝や指定した時刻に、今日の予定をまとめてリマインドしてほしい
* 予定ごとではなく、その日の予定全体をまとめて確認したい
* リマインド時刻を自分で設定したい
* 就活関連の予定を登録したい
* ES締切や面接予定を管理したい
* 締切が近い予定を確認したい
* 予定ごとにメモを残したい
* 予定のカテゴリを分けたい
* 予定の開始時刻、終了時刻、確認時刻を設定したい
* 自分のデータだけを管理したい
* 登録した予定を編集・削除したい
* 完了した予定やタスクを管理したい
* パスワードを忘れた場合、登録メールアドレスから再設定したい
* 自然文で入力した予定やタスクを、AIに整理してもらいたい
* AIが提案した予定やタスクを確認してから登録したい
* 就活予定や学習予定を、日付・時刻・カテゴリ付きで作成しやすくしたい

## 6. 利用シーン

### 6.1 今日の予定を確認するとき

1. ユーザーはログインする
2. トップページまたは今日の予定画面を開く
3. 今日登録されている予定やタスクを確認する
4. 締切が近い予定や確認時刻が近い予定を確認する
5. 必要に応じて予定を完了状態に変更する

### 6.2 予定を登録するとき

1. ユーザーは予定登録画面を開く
2. タイトル、カテゴリ、日付、開始時刻、終了時刻、確認時刻、メモを入力する
3. 登録ボタンを押す
4. 登録した予定が一覧画面や今日の予定画面に表示される

### 6.3 就活予定を登録するとき

1. ユーザーは予定登録画面を開く
2. カテゴリに「就活」を選択する
3. インターン応募、ES締切、面接日程、説明会などの予定を登録する
4. メモ欄に企業名、提出物、準備内容などを記録する
5. 今日の予定画面や日付ごとの予定画面で確認する

### 6.4 今日の予定をリマインドするとき

1. ユーザーはリマインド設定画面を開く
2. 今日の予定を受け取りたい時刻を設定する
3. システムは設定された時刻に、その日の予定一覧をまとめて通知する
4. ユーザーは通知内容を確認し、今日やるべき予定やタスクを把握する

### 6.5 パスワードを忘れたとき

1. ユーザーはログイン画面からパスワード再設定画面を開く
2. 登録済みメールアドレスを入力する
3. システムは再設定用リンクをメールで送信する
4. ユーザーはメール内のリンクから新しいパスワードを設定する
5. 新しいパスワードでログインできるようになる

### 6.6 AIで予定やタスクを作成するとき

1. ユーザーは自然文で予定やタスクを入力する
2. AIが入力内容から予定名、日付、時刻、カテゴリ、確認時刻、メモ、関連タスクを整理する
3. システムはAIが生成した予定案・タスク案を確認画面に表示する
4. ユーザーは内容を確認し、必要に応じて修正する
5. ユーザーが確定した場合のみ、予定またはタスクとして登録する

## 7. 解決したいこと

| 課題                 | 解決方針                          |
| ------------------ | ----------------------------- |
| 今日やるべきことが分かりにくい    | 今日の予定画面で当日の予定を一覧表示する          |
| 朝に今日の予定を確認し忘れる     | 指定した時刻に今日の予定をまとめてリマインドする      |
| 予定ごとの通知が多くて把握しづらい  | 1日の予定をまとめたリマインドを送る            |
| 締切が近い予定を見落としやすい    | 日付や確認時刻を登録し、近い予定を確認しやすくする     |
| 就活予定が分散しやすい        | 就活カテゴリを用意し、ES締切や面接予定をまとめて管理する |
| メモが分散しやすい          | 予定ごとにメモを保存できるようにする            |
| 予定登録に手間がかかる        | 将来的にAIが自然文から予定・タスク案を作成する      |
| 他人に予定を見られたくない      | ログインユーザーごとに予定データを分離する         |
| パスワードを忘れるとログインできない | メールによるパスワード再設定機能を用意する         |

## 8. スコープ

### 8.1 Version 1で作る範囲

Version 1では、予定管理アプリとして最低限使える状態を目指す。

* ユーザー登録
* ログイン
* ログアウト
* 予定登録
* 今日の予定表示
* 日付ごとの予定表示
* 予定詳細表示
* 予定編集
* 予定削除
* カテゴリ設定
* 確認時刻の設定
* リマインド時刻の設定項目
* 完了状態の管理
* ログインユーザーごとのデータ管理

### 8.2 Version 1では作らない範囲

Version 1では、以下の機能は実装対象外とする。

* Google Calendar連携
* メール通知
* LINE通知
* Discord通知
* 指定時刻での自動メール通知
* バックグラウンド処理による自動リマインド
* AIによる予定・タスク作成補助
* メールによるパスワード再設定
* 複数ユーザーでの予定共有
* スマートフォンアプリ化
* 完全な月間カレンダーUI
* 外部サービスとの本格的な連携

## 10. 完成条件

Version 1では、以下を満たすことを完成条件とする。

* ユーザー登録ができる
* ログインできる
* ログアウトできる
* ログインユーザーが予定を登録できる
* 登録した予定を今日の予定として確認できる
* 日付を選んで予定を確認できる
* 予定の詳細を確認できる
* 予定を編集できる
* 予定を削除できる
* 予定にカテゴリ、日付、開始時刻、終了時刻、確認時刻、メモを設定できる
* ユーザーがリマインド時刻を設定できる
* 予定の完了状態を管理できる
* ユーザーごとに予定データを分けて管理できる
* 他ユーザーの予定を閲覧・編集・削除できない

## 12. 制約条件

* 開発初期はPythonとDjangoを使用する
* データベースは開発初期ではSQLiteを使用する
* Version 1では外部通知や外部API連携は必須としない
* Version 1ではバックグラウンド処理による自動リマインドは必須としない
* AI機能は基本機能が完成してから追加する
* 個人情報や実際の就活データをGitHubに含めない
* パスワード、APIキー、メール認証情報などをGitHubに含めない
* 小さくても完成している状態を優先する

## 13. 成功基準

本アプリの成功基準は以下の通りである。

* ユーザーが予定を登録し、今日の予定として確認できる
* 就活、学習、開発などカテゴリ別に予定を管理できる
* ユーザーが朝や指定した時刻に今日の予定を確認しやすくなる
* 予定の確認漏れや締切漏れを減らせる
* ログインユーザーごとに安全にデータを分離できる
* 就活ポートフォリオとして、認証、CRUD、DB設計、日付・時刻処理、ユーザーごとのデータ管理を説明できる
* 将来的なAI機能、通知機能、外部サービス連携に拡張しやすい設計になっている
