# DayNavi 画面遷移図

## 1. 概要

DayNaviの画面遷移を整理する。

DayNaviは、日々の予定、タスク、就職活動のイベントを一元管理するWebアプリケーションである。
Version 1では、ユーザー登録、ログイン、予定管理、カテゴリ管理、今日の予定表示、リマインド時刻設定を中心に実装する。

画面遷移図はFigmaで作成し、画像として本ドキュメントに掲載する。

## 5. 主な画面遷移

### 5.1 未ログイン時

```text
トップページ
├── ユーザー登録画面
└── ログイン画面
```

未ログインユーザーは、トップページからユーザー登録またはログインに進む。

```text
ユーザー登録画面
└── 今日の予定画面
```

ユーザー登録が完了した場合、ログイン状態にして今日の予定画面へ遷移する。

```text
ログイン画面
└── 今日の予定画面
```

ログイン成功後、今日の予定画面へ遷移する。

---

### 5.2 ログイン後の基本遷移

```text
今日の予定画面
├── 予定一覧画面
├── 日付別予定画面
├── 予定登録画面
├── カテゴリ一覧画面
├── リマインド設定画面
└── ログアウト
```

ログイン後は、今日の予定画面を中心画面とする。
ユーザーは今日の予定を確認しながら、予定登録、予定一覧、カテゴリ管理、リマインド設定へ移動できる。

---

### 5.3 予定管理の画面遷移

```text
予定一覧画面
├── 予定登録画面
└── 予定詳細画面
    ├── 予定編集画面
    └── 予定削除確認画面
```

予定一覧画面では、自分が登録した予定だけを表示する。
予定詳細画面から、編集または削除確認に進む。

---

### 5.4 予定登録・編集・削除後の遷移

```text
予定登録画面
└── 今日の予定画面
```

予定登録後は、今日の予定画面へ遷移する。
登録した予定が今日以外の場合でも、今日の予定画面に戻る。

```text
予定編集画面
└── 予定詳細画面
```

予定編集後は、編集した予定の詳細画面へ遷移する。

```text
予定削除確認画面
└── 予定一覧画面
```

予定削除後は、予定一覧画面へ遷移する。

---

### 5.5 日付別予定の画面遷移

```text
今日の予定画面
└── 日付別予定画面
    └── 予定詳細画面
```

ユーザーは日付を指定して、その日の予定を確認できる。
日付別予定画面から予定詳細画面へ遷移できる。

---

### 5.6 カテゴリ管理の画面遷移

```text
今日の予定画面
└── カテゴリ一覧画面
    ├── カテゴリ登録画面
    ├── カテゴリ編集画面
    └── カテゴリ削除確認画面
```

ユーザーは自分の予定カテゴリを追加・編集・削除できる。
カテゴリはユーザーごとに管理し、他ユーザーのカテゴリは表示しない。

---

### 5.7 カテゴリ登録・編集・削除後の遷移

```text
カテゴリ登録画面
└── カテゴリ一覧画面
```

カテゴリ登録後は、カテゴリ一覧画面へ遷移する。

```text
カテゴリ編集画面
└── カテゴリ一覧画面
```

カテゴリ編集後は、カテゴリ一覧画面へ遷移する。

```text
カテゴリ削除確認画面
└── カテゴリ一覧画面
```

カテゴリ削除後は、カテゴリ一覧画面へ遷移する。

カテゴリを削除しても、登録済みの予定は削除しない。
カテゴリが削除された予定は、カテゴリ未設定として扱う。

---

### 5.8 リマインド設定の画面遷移

```text
今日の予定画面
└── リマインド設定画面
    └── 今日の予定画面
```

ユーザーはリマインド設定画面で、今日の予定を確認したい時刻を設定できる。
Version 1では、指定時刻での自動通知は行わず、設定値の保存とアプリ内表示を対象とする。

## 6. Version 1 全体遷移

Figmaでは、以下の構造を画面遷移図として作成する。

```text
トップページ
├── ユーザー登録画面
│   └── 今日の予定画面
└── ログイン画面
    └── 今日の予定画面
        ├── 予定一覧画面
        │   ├── 予定登録画面
        │   │   └── 今日の予定画面
        │   └── 予定詳細画面
        │       ├── 予定編集画面
        │       │   └── 予定詳細画面
        │       └── 予定削除確認画面
        │           └── 予定一覧画面
        ├── 日付別予定画面
        │   └── 予定詳細画面
        ├── 予定登録画面
        │   └── 今日の予定画面
        ├── カテゴリ一覧画面
        │   ├── カテゴリ登録画面
        │   │   └── カテゴリ一覧画面
        │   ├── カテゴリ編集画面
        │   │   └── カテゴリ一覧画面
        │   └── カテゴリ削除確認画面
        │       └── カテゴリ一覧画面
        ├── リマインド設定画面
        │   └── 今日の予定画面
        └── ログアウト
            └── トップページ
```

## 7. Version 2以降で追加する画面

Version 2では、パスワード再設定、月間カレンダー、検索・絞り込み、メールリマインドに関する画面を追加する。

| 画面名               | URL案                                | 概要                  |
| ----------------- | ----------------------------------- | ------------------- |
| パスワード再設定画面        | `/accounts/password_reset/`         | 登録メールアドレスを入力        |
| パスワード再設定メール送信完了画面 | `/accounts/password_reset/done/`    | 再設定メール送信後の案内        |
| 新パスワード設定画面        | `/accounts/reset/<uidb64>/<token>/` | メールリンクから新しいパスワードを設定 |
| パスワード再設定完了画面      | `/accounts/reset/done/`             | パスワード再設定完了          |
| 月間カレンダー画面         | `/calendar/month/`                  | 月単位で予定を表示           |
| 検索・絞り込み画面         | `/schedules/search/`                | 予定タイトル、カテゴリ、日付などで検索 |
| メールリマインド設定画面      | `/settings/email-reminder/`         | メール通知の有効化や通知時刻を設定   |

## 9. Version 3以降で追加する画面

Version 3では、AIによる予定・タスク作成補助機能を追加する。

| 画面名      | URL案                            | 概要               |
| -------- | ------------------------------- | ---------------- |
| AI予定作成画面 | `/ai/suggestions/new/`          | 自然文で予定やタスクを入力    |
| AI提案確認画面 | `/ai/suggestions/<id>/confirm/` | AIが作成した予定案を確認・修正 |
| AI提案履歴画面 | `/ai/suggestions/`              | 過去のAI提案履歴を表示     |

---

## 10. Version 3の画面遷移

### 10.1 AIによる予定作成

```text
今日の予定画面
└── AI予定作成画面
    └── AI提案確認画面
        ├── 予定登録完了
        │   └── 今日の予定画面
        └── 修正
            └── AI提案確認画面
```

AIが生成した予定・タスク案は、直接保存しない。
ユーザーが内容を確認し、確定した場合のみ予定として登録する。

---

### 10.2 AI提案履歴

```text
今日の予定画面
└── AI提案履歴画面
    └── AI提案確認画面
```

## 11. Figmaで作成する際のルール

Figmaでは、まず詳細なUIデザインではなく、画面名と遷移関係を整理する。

### 11.1 作成方針

* 各画面を四角形で表す
* 画面名を四角形の中に書く
* 矢印で画面遷移を表す
* Versionごとに色分けする
* ログインが必要な画面と不要な画面を区別する

### 11.2 色分け例

| 区分        | 用途             |
| --------- | -------------- |
| Version 1 | 基本機能           |
| Version 2 | 通知・検索・月間カレンダー  |
| Version 3 | AI機能           |
| 未ログイン画面   | トップ、ログイン、登録    |
| ログイン必須画面  | 予定管理、カテゴリ管理、設定 |

### 11.3 Figmaに配置する主なグループ

* 未ログイン時
* ログイン後メイン画面
* 予定管理
* カテゴリ管理
* リマインド設定
* Version 2追加画面
* Version 3追加画面

---

## 12. 備考

* Version 1では、今日の予定画面を中心に画面遷移を設計する
* 予定登録、編集、削除はログインユーザー本人の予定のみ対象とする
* カテゴリ管理はログインユーザー本人のカテゴリのみ対象とする
* カテゴリを削除しても、登録済み予定は削除しない
* Version 1では、指定時刻での自動通知や外部通知は実装しない
* Version 3のAI機能では、AIの提案内容をユーザーが確認してから登録する
